# 复习笔记智能助手 - 系统设计方案

> 目标：用户上传一门课程的 PPTX / PDF / DOCX / 图片，助手自动提取全部文字，整理输出一份**结构清晰、考试导向、带例题与试题附录**的 Markdown 复习笔记。
>
> 两条第一性原则：
> 1. **入口无损读取**：任何内容都不得因长度被截断或丢弃（否则无法判断什么考、什么不考）。
> 2. **出口考试优先**：复习资料不等于课程资料汇编——围绕"考什么/怎么考/什么最重要/什么最易错/如何速记"组织，优先复习价值而非内容覆盖率。

> 本文描述**已实现的原型**（`pipeline.py` 等），并在文末附录标注"完整版未实现"的能力。

---

## 〇、总体架构（5 阶段分治流水线）

不同类型资料的作用不同：课件是骨架、书/笔记用来查漏补缺、试题用来校准重要性和补例题。因此流水线**先按文件类型分类，再分阶段精修**，而不是把所有文件混在一起一把梭。

```
预处理：提取全部 records（extractors.py，逐文件逐页，带来源溯源）+ 去重
   |
   |-- 阶段1  分类（classify_files）
   |        每个文件读前 1-3 页 -> LLM 判 课件/书/笔记/试题/其他（失败回落启发式）
   |
   |-- 阶段2  课件建骨（phase_courseware）   仅 courseware 组
   |        标题解析（含续篇检测）-> 按标题分章 -> 每章分片生成
   |        （例题规则：课件有相关题可用，否则留 【例题：待补充】）-> 合并
   |        产物：draft[章节] = Markdown
   |
   |-- 阶段3  书/笔记精修（phase_refine）     仅 book+note 组
   |        按主题匹配片段 -> 对 draft 做 增/删/调/换
   |
   |-- 阶段4  试题校准（phase_exam）          仅 exam 组
   |        分析（真题 or 题库 + 高频/难/易 + 考试模式）-> 抽代表题入题库
   |        -> 每章按考试分析校准重要性/难度
   |
   |-- 阶段5  补例题（phase_fill_examples）
   |        按 课件 > 往年真题 > 题库 > 自编 的优先级，给 【例题：待补充】 填题
   |
   \-- 聚合 + 渲染（render_final）-> 复习笔记.md
        中间产物全部落盘到 work/，便于逐步检查
```

**阶段按需跳过**：无课件 -> 用书/笔记建骨（兜底）；无书/笔记 -> 跳过阶段3；无试题 -> 跳过阶段4。阶段5 只要存在占位就执行（无题源时自编兜底），保证最终输出无 `【例题：待补充】` 残留。

**代码落点**：编排与各阶段函数在 [pipeline.py](pipeline.py)；各阶段 prompt 在 [prompts.py](prompts.py)；提取在 [extractors.py](extractors.py)；大模型客户端在 [llm.py](llm.py)；入口在 [agent.py](agent.py) / [run.py](run.py)。

## 一、统一数据结构（贯穿全流程）

所有提取出的内容都表示为带**来源溯源**的 `Record`（`extractors.py`）：

```python
@dataclass
class Record:
    source_file: str       # 文件名（不含路径）
    source_type: str       # pptx | pdf | docx | image
    page: int              # 页码 / 幻灯片号 / 图片序号
    content: str           # 正文
    title: str = ""        # 所属标题（docx 标题链 / pptx 幻灯片标题）
    heading_level: int = 0 # docx 标题层级（0=正文，1/2/3=Heading）
```

> **关键设计：`source_file` + `page` 永远跟着内容走。** 它是试题附录能精确追溯到原始资料的基石，也是用户验证"有没有遗漏"的抓手。`chapter`/`is_problem` 等字段在原型里按需在阶段内推断，不持久化到 Record。

## 二、预处理：文件内容提取 + 去重

提取阶段**没有上下文限制**，必须把全文逐页落地，绝不截断。四种格式分别处理，输出都归一为 `Record`。

### 2.1 PPTX（`python-pptx`）
- 遍历每张幻灯片：标题占位符 -> `title`；正文文本框 -> `content`（按段落拼接）。
- **务必提取演讲者备注（notes_slide）**——备注里常有正文没有的完整解释和例题。
- 提取表格文本。一页幻灯片 = 一条 `Record`（`page` = 幻灯片号）。

### 2.2 PDF（`PyMuPDF/fitz`）
- 逐页 `page.get_text()`，一页文本 = 一条 `Record`。
- **扫描版/图片型 PDF 兜底**：若某页文本极少（< 阈值），判定为扫描页，渲染为图片走 2.4 的 OCR 分支。**这是 PDF"零遗漏"的关键防线。**

### 2.3 DOCX（`python-docx`）
- 遍历段落，按 `paragraph.style.name`（Heading 1/2/3...）建立**标题层级**，正文段落挂到最近的上级标题下；`title` 记录标题链（如 `"第三章 > 3.2 调度算法"`）。
- 提取表格（按行拼接为文本）。一段 = 一条 `Record`。

### 2.4 图片（`PaddleOCR`，中文场景优于 Tesseract）
- OCR 返回的文本块按**坐标位置重排**成阅读顺序，合并为该图的 `content`。
- 一图 = 一条 `Record`。未安装 PaddleOCR 时自动跳过并告警。

### 2.5 去重（`dedup_records`，两层）
| 层级 | 方法 | 用途 |
|------|------|------|
| 精确 | 归一化（去空白/标点/全半角）后哈希 | 抄录重复 |
| 子串 | 短串被长串包含则丢短 | 改写、嵌套段落 |

> 完整版的近似去重（MinHash/SimHash）与语义去重（句向量）见文末附录，原型未实现。

### 2.6 健壮性
- 文件加密 / 损坏 / 不支持的格式：跳过并写日志，**不中断**整批处理。

## 三、阶段1：文件分类（`classify_files`）

- 每个文件读前几页（`_preview`，约 1000 字），批量喂给大模型，要求返回严格 JSON `{文件名: courseware|book|note|exam|other}`。
- **解析容错（`_parse_json`）**：剥离 ```` ```json ```` 围裹、从首个 `{`/`[` 截取、坏 JSON 返回空。
- **启发式兜底（`heuristic_classify`）**：LLM 未返回或字段非法时回落——文件名+内容含 试卷/真题/题库/答案/试题 -> exam；pptx -> courseware；文件名含 lecture/handout/slide/课件/讲义 -> courseware（PDF 课件很常见）；含 笔记/重点/note -> note；其余 pdf/docx -> book。

---

## 四、阶段2：课件建骨（`phase_courseware`）— 全流程骨架

### 4.1 标题解析（`resolve_titles`，含续篇检测）
1. **启发式优先（`heuristic_title`）**：取该文件首个"第X章"标记，否则取 pptx 首个幻灯片标题或 docx 首个 H1。
2. **无标题文件交 LLM（`build_courseware_title_prompt`）**：给文件名 + 前几页 + 上一个课件的标题，判是否是续篇（`is_continuation`）。是 -> 沿用上一标题；否 -> 据内容生成新标题。

### 4.2 按标题分章（`group_courseware_chapters`）
- 按"文件标题"分组为章节；记录本身若含"第X章"标记则开启新章。续篇文件自然并入上一章。`--chapters` 显式指定时沿用显式列表。
- 无课件时走兜底 `gen_fallback_chapters`：用书/笔记按标题建章。

### 4.3 分片生成（`chunk_records` + `build_courseware_gen_prompt`）
- 每章按 `max_chars` 分片，相邻片保留 1 条 overlap（防边界概念被切半）。
- 逐片调模型生成本章复习资料，**例题规则**：课件里有相关题可引用（标注来源）；没有则留 `【例题：待补充】` 占位，本阶段不自编。
- **概念接续**：把前面片已得的概念名喂给后续片，命名保持一致，从源头减少同名碎片。
- 分片输出落 `work/10_ch{N}_c{k}.md`，合并（`_merge_chapter_parts`，去重复小标题）后落 `work/10_ch{N}.md`。

---

## 五、阶段3：书/笔记精修（`phase_refine` + `build_refine_prompt`）

对已成形的 draft 做**增/删/调/换**：
- **增**：课件遗漏或不考不重要但书/笔记强调的重要知识点，补进来。
- **删**：草稿里其实不考、不重要的内容，删掉或降为一句带过。
- **调**：按书/笔记调整重要性标记（【高频】/【重点】/【必背】）。
- **换**：纠正错误/不严谨的说法，或换更有代表性的表述/例题。

**主题匹配（`match_supplement`）**：用章名 + draft 小标题的 2-gram 关键词给每条书/笔记片段打分，取高分片段截断到预算喂给模型（原型级，未上向量）。约束：保留 `【例题：待补充】` 占位不动。产物落 `work/20_ch{N}.md`。书与笔记分两轮各跑一次。

---

## 六、阶段4：试题校准（`phase_exam` + `build_exam_analyze_prompt` / `build_calibrate_prompt`）

### 6.1 分析（分片 -> 合并，不截断）
试题按 `max_chars` 分片（`chunk_records`），逐片调 `build_exam_analyze_prompt` 出"分片分析"；多片时再调 `build_exam_merge_prompt` 合并去重为一份统一分析：**真题 or 题库**判定 + 高频/难点/易得分点 + 考试模式 + 每考点代表题。落 `work/30_exam_analysis.md`。**题库再大也不丢**（旧版只看前 40 条的截断已移除）。

### 6.2 候选题目（`collect_problems`，不全局截断）
收集全部候选题：`【课件题】`（`is_problem_record` 命中的课件片段）+ `【往年真题】`或`【题库题】`（按 6.1 判定）。清单落 `work/30_problems.json`。**不再取前 N**；交给阶段5 按章节相关匹配。

### 6.3 逐章校准
每章 draft + 考试分析 -> 调整重要性标记与先后顺序（高频/难点前置提级，易得分/不考降级后移），并在相关知识点后补"往年考法"提示。占位不动。产物落 `work/30_ch{N}.md`。

---

## 七、阶段5：补例题（`phase_fill_examples` + `build_fill_examples_prompt`）

把 draft 里的 `【例题：待补充】` 占位逐一替换为真实例题（题面 + 简要解答）。**选题优先级（从高到低）**：

```
课件题  >  往年真题原题  >  题库题  >  自编题（仅当上面都没有）
```

题目按**每章相关匹配**（`match_problems`：章名 + draft 小标题的 2-gram 关键词打分 + 优先级权重 课件题>往年真题>题库题），取相关高分题并截断到 `max_chars` 喂模型——**相关优先、按预算截断**，而非任意全局截断。池中无匹配则自编并标注（自编）。占位以外的内容与排版标记不变。产物落 `work/40_ch{N}.md`。无试题时仅含课件题或空（自编兜底），阶段5 仍执行。

---

## 八、分片机制（各阶段复用）

**核心原则：按"章节"切，而不是按固定字符窗切。** 按章节切让每片语义自洽、合并边界清晰。

```
章节字符预算 B = max_chars（默认 6000；长上下文模型可调大以减少分片数）
for chapter in chapters:
    按记录边界累加，超 B 则切片，相邻片保留 1 条 overlap（仅当上一条较小时，避免大段重复）
```

- **单条超长兜底（`_split_text` / `_split_record`）**：若某条 Record 本身 > B（如一页极密的教材 PDF），按段落 -> 句子 -> 字符逐级拆成多条 ≤ B 的子记录，溯源信息保留，**任何单条都不会撑爆单片**。
- **分片机制各阶段复用**：课件建骨、试题分析都走 `chunk_records`，所以大课件、大题库都不会被截断。
- **overlap**：防概念被切在边界、两片各看一半。
- 调用次数提示：N 章课程约 `1 + sum(各章分片数) + (书/笔记轮数)*N + (试题分片数+合并1) + N + N` 次，阶段跳过会相应减少。

## 九、最终输出（`render_final`，8 板块 Markdown）

```
# 《课程名》复习笔记
- 一、考试范围        （从资料抓 考试范围/题型/分值 关键句，找不到用章节兜底）
- 二、课程知识框架    （按章节缩进列表）
- 三、各章详解        （核心概念/原理/方法/易错点/题型/本章总结，用 【】标记）
- 四、全课程公式汇总  （从各章抽带编号或含 = 的短行）
- 五、全课程易错点汇总（收集 【易错点】 后的要点）
- 六、全课程高频考点汇总（收集 【必背/重点/高频】 后的要点）
- 七、考前速查手册    （每章一句话记忆 + 必背公式）
- 附：试题索引与代表题（有试题时，按 题目池 输出）
```

排版标记：`【定义】` `【高频】` `【重点】` `【必背】` `【易错点】` `【真题】` `【记忆】` `【解题步骤】`，公式单独成行并编号。聚合靠对标记的轻量正则解析（`extract_formulas` / `extract_easy_wrong` / `extract_high_freq`）。纯 Markdown，便于 Word/WPS/Typora 打开或转 PDF。

## 十一、关键设计点回顾

1. **提取与推理分离**：全文先落盘为 `Record`，提取不受上下文限制；模型只看分片。
2. **按文件类型分阶段**：课件建骨、书/笔记精修、试题校准、补例题各司其职，主次清晰。
3. **按章节分片 + overlap**：任何体积都能完整覆盖，边界不丢概念。
4. **例题两段式**：建骨阶段留占位、补例题阶段按优先级填，避免就地低质自编。
5. **阶段按需跳过 + 兜底**：缺哪类资料跳过哪阶段；无课件用书/笔记建骨；占位最终必被填。
6. **中间产物落 `work/`**：每阶段可逐步检查、回溯、调试。
7. **来源溯源贯穿全程**：每个知识点、每道题都能点回原始文件页码。
8. **分类与解析容错**：LLM 分类失败回落启发式；JSON 解析容错，流水线不崩。
9. **响应缓存与中断续跑**：每次调用按 `模型 + 系统提示 + 用户提示` 哈希缓存到 `work/cache.json`，未命中后立即落盘。中断后重跑同一命令，已完成调用自动命中跳过、不重新计费，只重做中断处及之后的（`--no-cache` 可强制全重跑）。缓存键含完整提示，故改 `input/` 文件只重算受影响章节；换模型需 `--no-cache` 或清缓存。

## 附录：完整版未实现能力（原型已留接口或可扩展）

- **实体消解与图合并**：跨章节同名/同义概念合并、关系边去重与冲突裁决（原型用 `_merge_chapter_parts` 去重复小标题近似）。
- **Mermaid 知识网络图**：核心概念及 contains/depends_on/parallel_to/contrast_with 关系的可视化（原型用章节缩进列表）。
- **结构化 JSON 输出**：每片输出 `{concepts, examples}`（含 aliases/importance/relations/source_refs），让合并机械化。原型为降复杂度采用 Markdown + 标记 + 轻量解析。
- **语义匹配**：书/笔记到章节、概念消解用句向量余弦（原型用关键词重叠）。
- **近似/语义去重**：MinHash、句向量（原型用归一哈希 + 子串包含）。
