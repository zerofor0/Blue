### 六、核心机制深度回顾

#### 重载（Overload）、重写覆盖（Override）与重写隐藏（Redefining）辨析 【必背】

##### 1. `override` 关键字 【必背】

【定义】
`override` 是 C++11 引入的关键字，用于在派生类中显式声明一个成员函数是对基类虚函数的重写覆盖。它向编译器明确表达程序员的意图，让编译器帮助检查函数签名是否与基类虚函数完全匹配。

原理与推导
在大型项目中，基类的虚函数签名可能发生变动，而派生类中原本意图重写覆盖的函数可能在无意间变成重写隐藏。`override` 通过编译期检查，强制要求被修饰的函数必须成功重写基类中的某个虚函数，否则产生编译错误，从而将运行时难以发现的逻辑错误提前暴露。

【解题步骤】
在派生类中声明重写覆盖函数时，在函数参数列表后、函数体前（或`const`/`final`等限定符之后）添加 `override` 关键字：
1. 确认基类中有对应签名的 `virtual` 函数。
2. 在派生类的同名函数声明末尾加上 `override`。
3. 若编译报错，检查：参数类型、`const` 限定符、返回类型是否与基类虚函数严格一致。

【易错点】

| 错误写法 | 正确写法 | 错误原因 |
| :--- | :--- | :--- |
| `void foo(float) override {}` | `void foo(int ) override {}` | 基类虚函数为 `foo(int)`，参数类型不同，`override` 要求严格匹配。 |
| `void bar() override {}` | // 不应使用 override | 基类中 `bar()` 不是虚函数，`override` 只能修饰重写虚函数的函数。 |

例题
**例题**：课件 [L8-虚函数-2026.pptx P37] 中 Derived3 的定义。
**题目**：
```cpp
class Base {
public:
    virtual void foo(int) { cout << "Base::foo(int)" << endl; }
    void bar() { cout << "Base::bar" << endl; }
};

class Derived3 : public Base {
public:
    // 请判断以下两个成员函数声明是否正确
    void foo(int ) override { cout << "Derived3::foo(int)" << endl; } // (A)
    // void foo(float ) override {}; // (B)
    // void bar() override {};      // (C)
};
```
**思路**：`override` 关键字要求该函数必须是对基类虚函数的一个合法重写覆盖。判断标准：(1) 基类必须有同名同参数的虚函数；(2) 参数类型、返回类型、`const`限定必须完全匹配。
**解答**：
- (A) 正确。`Base::foo(int)` 是虚函数，`Derived3::foo(int)` 签名完全一致，是对其的重写覆盖。
- (B) 错误。参数类型为 `float`，与基类 `foo(int)` 不匹配，且基类中无 `foo(float)` 虚函数，此时 `override` 要求失败，编译器报错。
- (C) 错误。`Base::bar()` 并非虚函数，`override` 只能修饰对虚函数的重写，编译器报错。
**技巧**：始终为派生类中意图重写虚函数的函数添加 `override` 关键字，它是免费的编译期保险。

##### 2. `final` 关键字 【重点】

【定义】
`final` 是 C++11 引入的关键字，有两个应用场景：
1. 修饰虚函数：表示该虚函数是最终版本，禁止后续派生类再次重写覆盖它。
2. 修饰类：表示该类是最终类，禁止被其他类继承。

原理与推导
`final` 提供了在继承链上“封堵”的能力。某些类的实现需要保证其行为不被修改（如安全相关、底层基础功能），使用 `final` 可以防止子类通过重写虚函数或进一步继承来改变既定行为，增强代码的稳定性和安全性。

【解题步骤】
1. **禁止函数被重写**：在虚函数声明末尾（`override` 之后）加上 `final`。
```cpp
class A : public Base {
    void foo() override final {} // 禁止后续派生类重写foo
};
```
2. **禁止类被继承**：在类名后、继承列表前加 `final`。
```cpp
class B final : public A {
    // ...
};
```

【易错点】

| 错误用法 | 正确用法 | 说明 |
| :--- | :--- | :--- |
| `void bar() final {}` | `virtual void bar() final {}` | `final` 确保函数为虚且不可被重写，修饰非虚函数会导致编译错误。 |
| `class Base { void foo() final; };` | `class Base { virtual void foo() final; };` | 同上，`foo` 必须首先是虚函数。 |

例题
**例题**：课件 [L8-虚函数-2026.pptx P40] 中的选择题。
**题目**：
下列说法中正确的一项是（ ）
A. 发生重写隐藏时，函数的参数必须相同。
B. 发生重写覆盖时，函数的参数必须相同。
C. 使用 `override` 修饰的函数可以不是虚函数。
D. 使用 `final` 修饰的函数可以不是虚函数。
**思路**：回顾三个核心概念的定义与限制。
**解答**：
- A 错误。重写隐藏发生的条件之一是参数不同，或基类函数非虚。
- B 正确。重写覆盖要求函数签名（包括函数名和参数类型）完全相同，返回类型协变。
- C 错误。`override` 强制该函数是对基类虚函数的重写，所以它本身就是虚函数。
- D 错误。`final` 明确要求被修饰的函数必须是虚函数。
正确答案为 B。
**技巧**：辨析题紧抓概念本质——覆盖看签名一致，隐藏因签名或虚性缺失，`override`/`final` 必饰虚函数。

##### 3. 重写覆盖的失效条件与协变返回类型 【高频】

【定义】
- **`const` 导致的覆盖失效**：成员函数的 `const` 限定符是函数签名的一部分。若基类虚函数为非 `const`，而派生类同名函数添加了 `const`，签名不同，无法构成重写覆盖，而是形成重写隐藏。
- **协变返回类型**：派生类重写虚函数时，若返回类型为基类虚函数返回类型的指针或引用，且两者为继承关系，也视为合法的重写覆盖。

原理与推导
C++ 将 `const` 等限定符视为函数签名组成，是保证语义一致性的需要。异步修改对象的函数与承诺不修改对象的函数本质不同。协变返回类型允许重写覆盖更精确地返回派生类类型，提升类型安全性，而不破坏多态机制。

【解题步骤】
1. 排查 `const` 覆盖陷阱：
   - 检查派生类意图重写函数的 `const` 限定是否与基类一致。
   - 若不一致，要么移除派生类的 `const`，要么用 `using Base::func;` 恢复被隐藏的函数。
2. 使用协变返回类型：
   - 派生类重写函数的返回类型必须是基类返回类型所在类的派生类的指针或引用。
   - 两者必须是同一类别（都是指针，或都是左值引用，或都是右值引用），不能是多级指针。

【易错点】
- 非 `const` 和 `const` 成员函数是完全不同的函数，很容易因疏忽造成“我以为我重写了”但实际是“隐藏”的错误。
- 协变仅适用于**指针**和**引用**返回类型。直接返回对象类型不是协变的，会导致编译错误。

例题
**例题**：课件 [L8-虚函数-2026.pptx P45] 中 `const` 影响覆盖的例子。
**题目**：
```cpp
class Base1 {
public: virtual void f() { cout << "Base1::f" << endl; }
};
class Derive1 : public Base1 {
public:
    void f() const { cout << "Derive1::f" << endl; } // 意图重写覆盖
    using Base1::f;
};
int main() {
    Derive1 a; const Derive1 b;
    a.f(); b.f();
    return 0;
}
```
**思路**：`Derive1::f() const` 与 `Base1::f()` 的函数签名因 `const` 限定不同而不同，构成重写隐藏。`using Base1::f;` 恢复基类函数可见性。`const` 对象与非 `const` 对象会匹配不同的重载版本。
**解答**：
- `a.f()`：`a` 是非常量对象，有 `Base1::f()` 和 `Derive1::f() const` 两个候选。非常量成员函数优先匹配非常量对象，因此调用 `Base1::f()`。
- `b.f()`：`b` 是常量对象，无法调用非 `const` 的 `Base1::f()`，只能调用 `Derive1::f() const`。
输出：
```
Base1::f
Derive1::f
```
**技巧**：检查虚函数重写时，必须核对函数名、参数列表、`const`/`volatile`/引用限定符等全套签名信息。
