### 七、正则表达式与C++实现

#### 正则表达式语法基础 【高频】

##### 【定义】正则表达式
正则表达式是由字母和符号组成的特殊文本，用于定义搜索文本时的一种规则模式。它可以极大地简化字符串匹配、查找和替换的复杂度。

##### 原理与推导
传统字符串校验方式（如 `check` 函数示例）需要对每个字符逐一判断，代码冗长且难以维护。正则表达式提供了一种声明式的模式匹配语言，通过特定的元字符组合来描述字符串的构成规则，将复杂的逻辑判断转化为模式字符串。

##### 【解题步骤】
1. 分析目标字符串的构成规则。
2. 将规则拆解为字符要求和重复次数。
3. 使用字符簇和重复模式组合成完整的正则表达式。

##### 字符簇与特殊字符
| 语法 | 含义 | 示例 |
|------|------|------|
| `[a-z]` | 匹配单个小写字母 | `[a-z]` 匹配 `a` 到 `z` 中的任意一个字符 |
| `[0-9]` | 匹配单个数字 | `[0-9]` 匹配 `0` 到 `9` 中的任意一个字符 |
| `[aeiou]` | 匹配指定集合中的任意一个字符 | 匹配任意一个元音字母 |
| `[^a-z]` | **取反**：匹配不在范围内的任意单个字符 | `[^c]ar` 匹配 `par`、`gar`，但不匹配 `car` |
| `\d` | 等价于 `[0-9]`，匹配单个数字 | `\d\d` 匹配 `12` |
| `\D` | 等价于 `[^0-9]`，匹配单个非数字 | `\D` 匹配 `a`、`B`、`_` |
| `\w` | 等价于 `[a-zA-Z0-9_]`，匹配字母、数字、下划线 | `\w+` 匹配 `hello_123` |
| `\W` | 等价于 `[^a-zA-Z0-9_]`，匹配非单词字符 | `\W` 匹配 `@`、` ` |
| `\s` | 匹配所有空白字符（空格、`\t`、`\n` 等） | `a\sb` 匹配 `a b` |
| `\S` | 匹配所有非空白字符 | `\S+` 匹配 `hello` |
| `.` | 匹配除换行符 `\n` 以外的任意单个字符 | `.ar` 匹配 `car`、`gar` |
| `\.` | 转义：匹配字面意义上的句号字符 `.` | `ge\.` 匹配 `ge.` |
| `^` | 匹配字符串的**开头** | `^The` 只匹配以 `The` 开头的字符串 |
| `$` | 匹配字符串的**结尾** | `bucket$` 只匹配以 `bucket` 结尾的字符串 |

##### 重复模式
| 语法 | 含义 | 示例 |
|------|------|------|
| `{n}` | 前面的内容恰好出现 `n` 次 | `a{4}` 匹配 `aaaa` |
| `{n,m}` | 前面的内容出现 `n` 到 `m` 次 | `a{2,4}` 匹配 `aa`、`aaa`、`aaaa` |
| `{n,}` | 前面的内容至少出现 `n` 次 | `a{2,}` 匹配 `aa`、`aaa`、... |
| `+` | 前面的内容至少出现 1 次（等价于 `{1,}`） | `a\w+` 匹配 `ar`、`arked` |
| `*` | 前面的内容出现 0 次或多次（等价于 `{0,}`） | `a\w*` 匹配 `a`、`ar`、`arked` |

##### 【易错点】
| 易混概念 | 区别与说明 |
|----------|------------|
| `[a-z]` vs `[a-z]+` | `[...]` 本身只匹配**单个**字符；必须配合 `+`、`*` 或 `{n,m}` 等才能匹配多个字符 |
| 通配符 `.` vs 正则 `.` | Shell 通配符中 `*` 代表任意字符串；正则中用 `.*` 表达类似含义 |
| `\` 转义冲突 | C++ 字符串中 `\` 也是转义符，写正则时必须用 `\\d` 或使用原生字符串 `R"(\d)"` |
| 大小写敏感 | `[a-z]` 不匹配大写字母，如果需要匹配建议使用 `[a-zA-Z]` 或设置忽略大小写标志 |
| 取反 `[^...]` | `[^0-9]` 匹配任意一个非数字字符，不是匹配"没有数字"，需要配合 `+` 使用 |

##### 【例题】
**例题**：用户名注册场景（课件 P84-85）
**题目**：设计一个正则表达式，校验用户名长度为3~15个字符，且只能包含小写字母、数字、下划线。合法：`john_123`；非法：`John_123` / `jo` / `@john`。
**思路**：拆解规则为：字符集 `[a-z0-9_]` 和长度限制 `{3,15}`，再加上字符串起止锚定符确保完全匹配。
**解答**：
```
^[a-z0-9_]{3,15}$
```
- `^` 和 `$` 确保整个字符串完全匹配，而非仅匹配子串。
- `[a-z0-9_]` 定义允许的单个字符范围。
- `{3,15}` 限制前面的字符集重复次数。
**技巧**：校验类场景必须加 `^` 和 `$`，否则只要存在合法子串就会误判通过。

#### C++正则表达式库 `<regex>` 【重点】

##### 正则表达式的三种模式
| 模式 | 含义 | 示例场景 |
|------|------|----------|
| **匹配 (Match)** | 判断**整个字符串**是否完全满足正则表达式 | 用户名格式校验 |
| **搜索 (Search)** | 在字符串中**查找**满足正则表达式的子串 | 提取文本中的数字串 |
| **替换 (Replace)** | 将匹配的子串**替换**为指定文本 | 敏感词过滤、格式化 |

##### 【定义】原生字符串
C++ 原生字符串语法 `R"(...)"` 取消转义，保留字符串的字面值。书写正则表达式时强烈推荐使用，避免双重转义。

```
"\\d+"  =  R"(\d+)"    // 都表示正则表达式 \d+
```

##### 关键组件与用法
| 功能 | 库组件 | 说明 |
|------|--------|------|
| 正则表达式对象 | `std::regex` | 保存编译后的正则表达式 |
| 匹配结果 | `std::smatch` | 存储匹配结果及捕获分组信息 |
| 完全匹配 | `std::regex_match(s, re)` | 字符串 `s` 是否**完全**匹配 `re` |
| 完全匹配（含捕获） | `std::regex_match(s, result, re)` | 匹配并存储结果到 `smatch result` |
| 搜索 | `std::regex_search(s, result, re)` | 搜索**第一个**匹配的子串 |
| 替换 | `std::regex_replace(s, re, fmt)` | 替换**所有**匹配子串为 `fmt`，返回新字符串 |

#### 捕获与分组 【必背】

##### 原理与推导
分组使用 `()` 将正则表达式的一部分括起来。匹配成功后，每个括号对应的子匹配内容会被"捕获"并存储在 `smatch` 对象中，便于提取结构化的信息片段。

##### 【解题步骤】
1. 在正则表达式中用 `()` 包裹需要提取的部分。
2. 调用 `regex_match` 或 `regex_search`，传入 `smatch` 对象。
3. 通过 `sm[i]` 访问第 `i` 个分组内容。
   - `sm[0]`：始终是完整匹配的字符串。
   - `sm[1]`：第一个括号分组的内容。
   - `sm[2]`：第二个括号分组的内容，依此类推。

##### 【易错点】
| 错误写法/理解 | 正确写法/理解 |
|---------------|----------------|
| 认为 `sm[0]` 是第一个括号的内容 | `sm[0]` 是完整匹配，`sm[1]` 才是第一个括号分组 |
| `regex_match` 和 `regex_search` 混用 | `match` 要求**全串**匹配；`search` 只要求**存在子串**匹配 |
| 连续搜索时忘记更新搜索范围 | 使用 `sm.suffix().str()` 获取剩余未搜索的部分作为下一次搜索的字符串 |

##### 替换中的分组引用
在 `regex_replace` 的替换文本中，可使用特殊符号引用捕获分组：
- `$&`：代表整个匹配的子串。
- `$1`, `$2`, ...：代表第 1、2 个捕获分组的内容。

**示例**：`regex_replace(s, regex(R"((sub)([\S]*))"), "[$&]")` 将每个匹配的单词用方括号括起来。

#### 综合应用示例：学生信息整理 【高频】

##### 背景与分析
从一段自我介绍文本中分别提取：姓名、出生日期、电话号码、邮箱。各信息特征如下：
- **姓名**：以 "I am xxx." 或 "My name is xxx." 开头。
- **出生日期**：`yyyy-mm-dd` 或 `yyyy.mm.dd` 格式。
- **电话**：11位数字，不以0开头。
- **邮箱**：由数字、字母、`@`、`.` 组成。

##### 【解题步骤】
1. 分别为每个信息项编写对应的正则表达式。
2. 对同一输入字符串分别执行 `regex_search` 提取各项。
3. 对于日期，利用捕获分组分别获取年、月、日，并进行格式化（如补零）。
4. 按要求格式输出所有提取到的信息。

##### 关键正则表达式设计
| 信息项 | 正则表达式 | 说明 |
|--------|-----------|------|
| 姓名 | `(My name is \|I am )(\w+)\.` | 捕获第二个分组 `sm[2]` 得到姓名 |
| 生日 | `(\d{4})[\.-](\d{1,2})[\.-](\d{1,2})` | 捕获三个分组：年、月、日 |
| 电话 | `[1-9]\d{10}` | 首位1-9，后跟10位数字，共11位 |
| 邮箱 | `[\w]+@(\w+\.)+\w+` | 匹配 `xxx@yyy.zzz` 格式，分组1捕获主域名部分 |

##### 例题：【课件 P103-105】
**例题**：学生信息提取
**题目**：
输入字符串：
```
I am zhangshuaishuai. I was born on 2000.10.2. My phone number is 18866667777 and you can also reach me by my email: zhangss@tsinghua.edu.cn
```
提取姓名、生日（格式 `yyyy.mm.dd`，月日不足两位补零）、电话号码、邮箱并输出。

**思路**：
1. 分别定义姓名、日期、电话、邮箱的正则表达式对象。
2. 使用 `regex_search` 逐个提取信息，利用 `smatch` 获取捕获分组。
3. 日期部分使用 `stoi` 将捕获的年月日字符串转为整数，再用标准输出控制格式（本例中课件代码直接输出 `sm[i]` 未补零，实际生产环境中可按需格式化）。

**解答**：
```cpp
void extract(string input) {
    smatch sm;
    regex get_name(R"((My name is |I am )(\w+)\.)");
    regex get_date(R"((\d{4})[\.-](\d{1,2})[\.-](\d{1,2}))");
    regex get_mobile(R"([1-9]\d{10})");
    regex get_email(R"([\w]+@(\w+\.)+\w+)");

    if (std::regex_search(input, sm, get_name))
        cout << sm[2] << endl;
    
    int date[3] = {0};
    if (regex_search(input, sm, get_date)){
        for (int i = 1; i <= 3; i++)
            date[i - 1] = stoi(sm[i]);
        cout << date[0] << "." << date[1] << "." << date[2] << endl;
    }
    
    if (regex_search(input, sm, get_mobile))
        cout << sm[0] << endl;
    
    if (regex_search(input, sm, get_email))
        cout << sm[0] << endl;
}
```
**技巧**：姓名正则 `(\w+)\.` 依赖句子结尾的句号作为分隔，实际应用需注意文本格式稳定性；电话正则已通过首位 `[1-9]` 排除了以 0 开头的号码。
