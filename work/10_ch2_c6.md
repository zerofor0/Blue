### 四、函数对象（Function Object）回顾

#### std::function类型擦除机制 【必背】

##### 1. 【定义】
`std::function` 是C++标准库提供的通用多态函数包装器，可以存储、复制和调用任何可调用目标（函数、函数指针、函数对象、lambda表达式等）。它通过 **类型擦除（Type Erasure）** 机制，屏蔽不同可调用类型的差异，提供统一的接口。

- **声明形式**：`std::function<返回类型(参数类型列表)>`
- 例：`std::function<int(int, int&)>` 表示存储一个返回`int`，接受`int`和`int&`两个参数的可调用对象。

##### 2. 原理与推导：类型擦除与兼容性准则

`std::function` 内部通常使用一个模板化的派生类继承自一个非模板的基类，将具体类型信息“擦除”在基类接口之后。关键在于其 **绑定兼容性准则**，该准则源于调用安全性：

> `function` 参数必须比实际参数 **更严格**，`function` 返回值必须比实际返回值 **更宽松**。

- **逻辑推导**：
  1. **参数准则**：所有能被 `std::function` 接受的参数，都必须能传递给实际的可调用对象。因此，`function` 上声明的参数类型必须隐含地向实际参数类型转换是安全的。通常，这意味着 `function` 的参数类型（接受端）不能比实际参数（要求端）更“宽松”。
      - **更严格** 的含义：例如，接受 `int`、`const int&` 比接受 `int&` 更严格，因为 `int` 和 `const int&` 能接收右值或常量，而 `int&` 只能绑定左值。
  2. **返回值准则**：所有实际可调用对象可能返回的值，都必须能被 `std::function` 的返回类型所包容。因此，`function` 的返回类型不能比实际返回类型更“严格”。
      - **更宽松** 的含义：例如，实际返回 `int&`，`function` 返回 `int` 是安全的（可以拷贝引用），反之则不安全（可能绑定到临时对象）。

- **绑定优先级（严格程度）**：
根据PPT第122页，对于参数绑定而言，其严格程度从高到低（从松到严的理解顺序相反）遵循：
  - **左值**（`T&`） < **常量左值**（`const T&`） < **常量右值**（`const T&&`） < **右值**（`T&&`）
  - 不带引用/const的普通值类型（如`int`）在参数传递中，其行为最宽松，可包容左值、右值等多种情况。在准则中，它比带引用的类型更“严格”（此处“严格”指对传入`function`的实参限制更严，更安全）。

##### 3. 【解题步骤】`std::function` 兼容性判断
用于判断 `function<Fx(Args...)>` 能否绑定可调用对象 `Callable`。

1.  **提取类型**：明确 `Callable` 的实际参数类型 `P` 和返回值类型 `R`，以及 `std::function` 声明的参数类型 `FArgs` 和返回值类型 `FRet`。
2.  **检查参数**：对每一对 `FArg`（来自 `FArgs`） 和 `PArg`（来自 `P`），必须满足：`FArg` 能安全地传递给 `PArg`。根据上述原则，`FArg` 应比 `PArg` 更严格。可查询绑定优先级表。
3.  **检查返回值**：必须满足：`R` 能安全地赋值/转换为 `FRet`。根据原则，`FRet` 应比 `R` 更宽松。
4.  **特别例外**：当实际参数类型为 `T&&`（右值引用）时，`std::function` 的参数可以为 `T`。因为 `std::function` 内部会拷贝实参生成一个左值副本，然后以右值形式传给实际函数。这是唯一允许 `function` 参数比实际参数宽松的情况。
5.  **结论**：参数与返回值均满足上述要求，则可编译通过。

##### 4. 【易错点】

| 易混概念 | 辨析 | 正例/反例 |
| :--- | :--- | :--- |
| **函数声明 vs `std::function` 包装** | 对于函数 `int func(const int& x, int& b)`，调用时 `func(int_val, int_ref)` 是合法的。但在 `std::function` 包装时，必须严格遵循兼容性准则。 | 允许：`function<int(int, int&)>`<br/>不允许：`function<int(int, int)>` （第二参数 `int` 比 `int&` 宽松） |
| **`function` 参数 `T` vs `T&`** | `function` 用 `T&` 比 `T` 更宽松（限制少），因为 `T&` 只能接左值，而 `T` 能接左值和右值。因此，在准则中 `T` 比 `T&` 更严格。 | 实际参数为 `int&`，则 `function` 参数可为 `int&` 或 `int`，但若实际参数为 `int`，`function` 参数为 `int&` 则可能失败。 |
| **`function` 返回值 `T` vs `T&`** | `function` 用 `T` 比 `T&` 更宽松（包容性强），因为返回 `T` 时，无论实际返回 `T` 还是 `T&` 都可以拷贝。返回 `T&` 更严格。 | 实际返回 `int&`，`function` 返回 `int` （正确） 或 `int&` （正确）。实际返回 `int`，`function` 返回 `int&` 则错误。 |
| **右值参数（`T&&`）的例外** | 当实际参数为 `T&&` 时，准则存在例外，允许 `std::function` 参数为 `T`。其内部通过拷贝参数来安全地生成右值。 | `int& operator()(int&&)` 可被 `function<int&(int)>` 包装。 |

##### 5. 例题

**例题**：来自课堂拓展自学题 [L11-iostream&函数对象和智能指针-2026.pptx P112-P118]

**题目**：给定函数 `int func1(const int &x, int &b) {...}`，对以下 `std::function` 声明进行判断，哪些能成功编译并绑定 `func1`？
A. `function<int(int, int)> pf1 = func1;`
B. `function<int(int, int&)> pf1 = func1;`
C. `function<int(int&, int&)> pf1 = func1;`
D. `function<int(const int&, int&)> pf1 = func1;`
E. `function<int(const int, int)> pf1 = func1;`
F. `function<int&(int, int&)> pf1 = func1;`

**思路**：应用 `std::function` 兼容性准则。`func1` 实际参数为 `(const int&, int&)`，返回值为 `int`。逐项检查每个选项的参数对和返回值对是否满足“function参数更严格，function返回值更宽松”。

**解答**：
1.  **A.** 第二参数：`function` 的 `int` 比实际的 `int&` 更宽松，不满足参数准则。 ×
2.  **B.** 第一参数：`int` 和 `const int&` 范围一致（都可靠接右值/左值）。第二参数：`int&` 和 `int&` 一致。返回值：`int` 和 `int` 一致。所有条件满足。 **√**
3.  **C.** 第一参数：`function` 的 `int&` 比实际的 `const int&` 更严格（`int&` 只能绑左值），满足准则。第二参数：匹配。返回值匹配。 **√**
4.  **D.** 所有参数类型与返回值类型完全一致。 **√**
5.  **E.** 第二参数：`function` 的 `int` 比实际的 `int&` 更宽松，不满足参数准则。 ×
6.  **F.** 返回值：`function` 的 `int&` 比实际的 `int` 更严格（`function` 返回引用，但实际可能返回临时量），不满足返回值准则。 ×

**技巧**：判断 `std::function` 绑定的口诀：`function`参数的接口要“吝啬”（限制严），`function`返回值的接口要“慷慨”（包容性强）。遇到右值引用参数，立即想到内部拷贝例外。
