# 课程设计选题报告：复习笔记智能助手

## 一、计划开发的 Agent

本项目计划开发一款面向高校学生的复习资料整理 Agent。用户上传一门课程的课件、教材、笔记与试题等资料，Agent 自动提取全部文字内容，按考试导向整理并输出结构化的复习笔记。

## 二、解决的实际问题

学生期末复习存在三个主要问题：一是资料格式多样、数量大，人工整合耗时；二是直接交由大模型处理时受上下文长度限制，内容易被截断；三是难以从资料中准确识别考点与重要程度。本 Agent 通过多格式提取、按章节分片处理、按资料类型分阶段整理，解决资料整合、内容遗漏与考点识别问题。

## 三、计划包含的功能

核心功能：

- 多格式资料提取：支持 PPTX、PDF、DOCX、图片（OCR），保留来源文件与页码。
- 资料自动分类：识别课件、教材、笔记、试题、其他五类。
- 分阶段整理：课件构建章节框架，教材与笔记补充修订，试题校准重要性与难度，按优先级补充例题。
- 长文本分片处理：按章节分片，超长内容自动拆分，避免内容截断。
- 断点续跑：缓存中间结果，处理中断后可从断点继续。
- 结构化输出：生成包含考试范围、知识框架、各章详解、公式汇总、易错点、高频考点、速查手册、试题附录的复习笔记。

扩展功能：

1. 格式学习：支持用户提供样例笔记，Agent 据此调整输出排版与风格。
2. 排版与 PDF 输出：增加目录、页眉页脚与公式排版，导出 PDF。
3. 输出长度控制：支持设定总页数或每章字数上限。
4. 聊天式精修：生成后用户以自然语言提出修改要求，Agent 增量更新。
5. 图形化界面：提供文件上传、进度显示、参数配置与结果预览。
6. 程序打包：打包为 .exe，脱离 Python 环境运行。

## 四、技术方案

资料提取拟采用 python-pptx、PyMuPDF、python-docx、PaddleOCR；模型调用拟采用 OpenAI 兼容接口（GLM、DeepSeek、通义千问等）；结果输出为 Markdown，经 Pandoc 转为 PDF；图形界面拟采用 PyQt 或 Tkinter；程序打包拟采用 PyInstaller。所选技术均为成熟开源方案，可行性强。
